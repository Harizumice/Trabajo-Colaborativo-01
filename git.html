COMANDOS EN GIT
Nombre del comando Para qué se sirve
git --version muestra qué versión de git estamos usando.
git config --help sirve por si necesitamos ayuda con comandos.
git config --global user.name "NombreUsuario" registra nuestro nombre para el repositorio.
git config --global user.email "correo2023@gmail.com" registra nuestro correo electrónico para el repositorio.
git config -l Es para comprobar la información del repositorio, entre ella, nuestro inicio de sesión.
git init sirve para crear el repositorio, sólo es necesario hacerlo una vez dentro de la respectiva carpeta que tenga un proyecto.
git add . Es para añadir los archivos o cambios realizados del proyecto en el repositorio.
git commit -m "Esto es un comentario para el repositorio" Sirve para colocar comentarios luego de añadir los cambios al repositorio.
ls sirve para ver los archivos o carpetas que hay dentro de la carpeta en la que hayamos iniciado el git bash.
cd NombreCarpeta/desktop Es para dirigirnos a una carpeta/escritorio.
mkdir NombreCarpeta sirve para crear una carpeta (importante no usar espacio, en su lugar se puede usar un "_" o "-")
ls -la sirve para verificar si la página está vacía o no.
start . Es para abrir la carpeta en la que nos encontremos dentro del git bash.
cd .. Es para devolvernos a la carpeta anterior de la actual.
code . Inicia el Visual Studio Code desde la carpeta en la que estemos.
touch NombreArchivo Crea nuevos archivos en la carpeta. Para hacer varios con el mismo comando se pone espacio.
git status Muestra el estado actual del repositorio
git log Muestra los commits/comentarios que hemos hecho en el repositorio y quién los ha hecho
git checkout -f MADE IN HEAVEN! (Si quieres reiniciar los cambios de golpe úsalo)
git diff --stat NombreArchivo.etiqueta Muestra en detalle las modificaciones de un archivo del repositorio.
